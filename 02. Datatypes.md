# JavaScript Data Types

> "Everything in JavaScript is a value, and every value has a data type."

Understanding data types is one of the most important concepts in JavaScript because **every operation depends on the type of the value**.

---

# What is a Data Type?

A data type defines:

- What kind of value is stored.
- How much memory is needed.
- What operations can be performed on it.

Example

```javascript
let age = 20;
```

Here,

```
20
```

is a **Number**.

Example

```javascript
let name = "Madan";
```

Here,

```
"Madan"
```

is a **String**.

---

# Why Do We Need Data Types?

Imagine JavaScript didn't know the type.

```javascript
10 + 20
```

Should it perform

```
Addition
```

or

```
String Concatenation
```

?

Now compare

```javascript
"10" + "20"
```

Output

```
1020
```

Since JavaScript knows the type, it performs the correct operation.

---

# Types of Data Types

JavaScript has **8 data types**.

They are divided into two categories.

```
Data Types
│
├── Primitive
│
└── Non-Primitive (Reference)
```

---

# Primitive Data Types

Primitive values are immutable.

This means they **cannot be changed** after they are created.

Instead, JavaScript creates a **new value**.

There are **7 primitive types**.

```
1. Number
2. String
3. Boolean
4. Undefined
5. Null
6. Symbol
7. BigInt
```

---

# Non-Primitive Data Types

Everything that is **not primitive** is an object.

Examples

```
Object
Array
Function
Date
Map
Set
RegExp
```

Internally,

All of these are objects.

---

# Primitive vs Reference

```
Primitive

Variable
   │
   ▼
 Value
```

Example

```javascript
let x = 10;
```

Memory

```
x
│
▼
10
```

---

Reference

```
Variable
   │
   ▼
Memory Address
   │
   ▼
Object
```

Example

```javascript
let user = {

    name: "Madan"

};
```

Memory

```
user
 │
 ▼
0x100

↓

{
 name: "Madan"
}
```

The variable stores the reference, not the object itself.

Represents

- Integers
- Floating-point numbers
- Infinity
- -Infinity
- NaN

Examples

```javascript
let age = 20;

let salary = 35000.75;

let temperature = -15;
```

All are Number.

---

Special Number Values

## Infinity

```javascript

```

Output

```
Infinity
```

---

## -Infinity

```javascript
console.log(-10 / 0);
```

Output


---

## NaN

Means

```
Not a Number
```

Example

```javascript
console.log("Hello" * 5);
```

Output

```
NaN
```

NaN is surprisingly of type Number.

```javascript
typeof NaN
```

Output

```

```

---

# String

Represents text.

Can be written using

```javascript
"Hello"

'Hello'

`Hello`
```

Examples

```javascript
let name = "Madan";

let city = 'Chennai';

let msg = `Hello`;
```

---

String Concatenation

```javascript
let first = "Madan";

let last = "Kumar";

console.log(first + " " + last);
```

Output

```
Madan Kumar
```

---

Template Literals

```javascript
let name = "Madan";

console.log(`Hello ${name}`);
```

Output

```
Hello Madan
```

---

# Boolean

Represents only two values.

```
true

false
```

Example

```javascript
let isLoggedIn = true;

```

Mostly used in

- Conditions
- Loops
- Comparisons

---

# Undefined

Means

```
Value has not been assigned yet.
```

Example

```javascript
let age;

console.log(age);
```

Output

```
undefined
```

Notice

The variable exists.

It simply has no value.

---

# Null

Represents

```
Intentional absence of value.
```

Example

```javascript
let user = null;
```

Meaning

```
Currently no user.
```

Difference

```
undefined

↓

JavaScript gives it.

-----------------------

null

↓

Programmer gives it.
```

---

# Symbol

Introduced in ES6.

Used to create unique identifiers.

Example

```javascript
const id1 = Symbol();

const id2 = Symbol();

console.log(id1 == id2);
```

Output

```
false
```

Every Symbol is unique.

Mostly used in advanced programming and libraries.

---

Example


Notice

```
n
```

This creates a BigInt.

Type


# Primitive vs Reference Copy

Primitive

```javascript
let a = 10;

let b = a;

b = 20;
```

Memory

```
a → 10

b → 20
```

Changing one does not affect the other.

---

Reference

```javascript
let user1 = {

    name: "Madan"

};

let user2 = user1;

user2.name = "John";
```

Memory

```
user1
   │
   ▼
Object

▲
│

user2
```

Both variables point to the same object.

Output

```javascript
console.log(user1.name);
```

```
John
```

---

# Mutable vs Immutable

Primitive

```
Immutable
```

Cannot change.

Example

```javascript
let name = "Madan";

name = "John";
```

JavaScript creates a new string.

---

Objects

```
Mutable
```

Example

```javascript
let user = {

    age:20

};

user.age = 25;
```

Same object.

Property changed.

# Golden Rules

- Everything in JavaScript is a **value**.
- JavaScript has **8 data types**.
- **7 are primitive** and **1 is non-primitive (Object)**.
- Primitive values are **immutable**.
- Objects are **mutable**.
- Arrays and functions are **objects internally**.
- `typeof null` returning `"object"` is a historical bug.
- Use `Array.isArray()` to check arrays.
- Use `typeof` to determine the type of a value.
