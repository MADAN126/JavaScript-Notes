# JavaScript Data Types

> **Everything in JavaScript is a value. Every value has a type.**
>
> JavaScript decides **how to store a value, how much memory to allocate, and what operations are allowed** based on its data type.

---

# Why Data Types Exist

Without data types, JavaScript cannot decide what an operation means.

```javascript
10 + 20
```

Should it perform:

- Addition (`30`)
- String concatenation (`1020`)

Now compare:

```javascript
"10" + "20"
```

Output

```text
1020
```

**Rule:** The type of the operands decides the behavior.

---

# What is a Data Type?

A **data type** tells JavaScript:

- What the value represents.
- How it is stored in memory.
- What operations are valid.

Example

```javascript
let age = 20;
```

```
20 → Number
```

```javascript
let name = "Madan";
```

```
"Madan" → String
```

---

# JavaScript Data Types

JavaScript has **8 data types**.

```
Data Types
│
├── Primitive (7)
│
└── Object / Reference (1)
```

---

# Primitive Data Types

Primitive values represent **a single immutable value**.

When a primitive changes, JavaScript **creates a new value** instead of modifying the old one.

```
Number
String
Boolean
Undefined
Null
Symbol
BigInt
```

### Memory Model

```
Variable
   │
   ▼
 Value
```

```javascript
let x = 10;
```

```
x
│
▼
10
```

Every primitive variable stores its own value.

---

# Reference (Object) Type

Everything that is **not primitive** is an object.

Examples

```
Object
Array
Function
Date
Map
Set
RegExp
```

### Memory Model

```
Variable
   │
   ▼
Memory Address
   │
   ▼
Object
```

```javascript
let user = {
    name: "Madan"
};
```

```
user
 │
 ▼
0x100
 │
 ▼
{
 name:"Madan"
}
```

The variable **does not store the object**.

It stores **the address of the object**.

---

# Primitive vs Reference

| Primitive | Reference |
|-----------|-----------|
| Stores value | Stores address |
| Immutable | Mutable |
| Copied by value | Copied by reference |
| Independent copies | Shared object |

# Primitive Copy

```javascript
let a = 10;

let b = a;

b = 20;
```

Memory

```
a → 10

b → 20
```

Changing one variable **does not affect** the other.

---

# Reference Copy

```javascript
let user1 = {
    name: "Madan"
};

let user2 = user1;

user2.name = "John";
```

Memory

```
user1 ─────┐
           │
           ▼
        Object
           ▲
           │
user2 ─────┘
```

Output

```javascript
user1.name
```

```
John
```

Both variables point to the **same object**.

# Mutable vs Immutable

## Primitive

```
Immutable
```

```javascript
let name = "Madan";

name = "John";
```

A new string is created.

---

## Object

```
Mutable
```

```javascript
const user = {
    age:20
};

user.age = 25;
```

Same object.

Only its property changes.

# Golden Rules

- Everything in JavaScript is a **value**.
- Every value has exactly **one data type**.
- JavaScript has **8 data types**.
- **7 are primitive**, **everything else is an object**.
- Primitive variables store **values**.
- Objects store **references (memory addresses)**.
- Primitive values are **immutable**.
- Objects are **mutable**.
- Arrays are objects.
- Functions are callable objects.
- `typeof null === "object"` is a historical bug.
- Use `Array.isArray()` to detect arrays.
- Use `typeof` for basic type checking.
