# Writing JavaScript in HTML

> **JavaScript is written inside HTML using the `<script>` tag.**

The browser reads the HTML first, and whenever it encounters a `<script>` tag, it executes the JavaScript code inside it.

---

# Methods to Add JavaScript to HTML

There are **three** ways.

```
JavaScript in HTML

│

├── Inline JavaScript

├── Internal JavaScript

└── External JavaScript
```

---

# 1. Inline JavaScript

JavaScript is written directly inside an HTML element using an event attribute.

Example

```html
<!DOCTYPE html>
<html>

<body>

    <button onclick="alert('Hello')">
        Click Me
    </button>

</body>

</html>
```

Output

```
Click button

↓

Alert appears

↓

Hello
```

### How it works

```
User clicks button

↓

onclick event fires

↓

Browser executes JavaScript

↓

alert("Hello")
```

### Why it's not recommended

- Mixes HTML and JavaScript.
- Difficult to maintain.
- Repeats code.
- Hard to debug.

Use only for very small examples.

---

# 2. Internal JavaScript

JavaScript is written inside the `<script>` tag in the same HTML file.

Example

```html
<!DOCTYPE html>
<html>

<head>

    <title>JavaScript</title>

</head>

<body>

    <h1>Hello</h1>

    <script>

        console.log("Welcome");

    </script>

</body>

</html>
```

Output (Console)

```
Welcome
```

### Execution Flow

```
Browser reads HTML

↓

Reads <h1>

↓

Reaches <script>

↓

Executes JavaScript

↓

Continues reading HTML
```

---

# 3. External JavaScript (Recommended)

Write JavaScript in a separate `.js` file.

**index.html**

```html
<!DOCTYPE html>
<html>

<head>

    <title>JavaScript</title>

</head>

<body>

    <h1>Hello</h1>

    <script src="script.js"></script>

</body>

</html>
```

**script.js**

```javascript
console.log("Welcome");
```

Output

```
Welcome
```

### Why external JavaScript is best

- Keeps HTML clean.
- Reusable across multiple pages.
- Easier to maintain.
- Easier to debug.
- Faster loading due to browser caching.

This is the standard approach used in real-world projects.

---

# The `<script>` Tag

Basic syntax

```html
<script>

    // JavaScript code

</script>
```

Or

```html
<script src="script.js"></script>
```

### Important Rule

When using `src`, **do not** place JavaScript code inside the `<script>` tag.

Wrong

```html
<script src="script.js">

console.log("Hello");

</script>
```

The browser ignores the code inside.

Correct

```html
<script src="script.js"></script>
```

---

# Where Should `<script>` Be Placed?

## Method 1 — End of `<body>` (Recommended)

```html
<body>

    <h1>Hello</h1>

    <script src="script.js"></script>

</body>
```

### Why?

The browser loads HTML first.

```
HTML

↓

DOM Created

↓

JavaScript Executes
```

The DOM is ready when your JavaScript runs.

---

## Method 2 — Inside `<head>`

```html
<head>

    <script src="script.js"></script>

</head>
```

Problem

The browser executes JavaScript **before** creating the HTML elements.

Example

```javascript
document.getElementById("title");
```

If the HTML isn't loaded yet,

```
null
```

is returned.

---

# Using `defer`

A better way to load scripts in the `<head>`.

```html
<head>

    <script src="script.js" defer></script>

</head>
```

Execution

```
Browser Downloads JS

↓

Continues Parsing HTML

↓

DOM Finished

↓

Runs JavaScript
```

This is the modern recommended approach.

---

# Using `async`

```html
<script src="script.js" async></script>
```

Execution

```
Download JS

↓

Execute Immediately

↓

Continue HTML
```

The script may run **before the DOM is fully built**, so use `async` only when the script does not depend on the page structure.

---

# `defer` vs `async`

| Feature | `defer` | `async` |
|---------|---------|---------|
| Downloads in parallel | ✅ | ✅ |
| Waits for HTML parsing | ✅ | ❌ |
| Executes in order | ✅ | ❌ |
| Best for app scripts | ✅ | ❌ |
| Best for independent scripts (analytics, ads) | ❌ | ✅ |

# Writing Multiple Scripts

```html
<script>

console.log("One");

</script>

<script>

console.log("Two");

</script>
```

Output

```
One

Two
```

Scripts execute from top to bottom.

---

# Multiple External Files

```html
<script src="first.js"></script>

<script src="second.js"></script>
```

Execution

```
first.js

↓

second.js
```

---

# Mixing Internal and External JavaScript

```html
<script>

</script>

<script src="script.js"></script>
```

Execution order

```
Internal

↓

External
```

# Connecting HTML with JavaScript

**index.html**

```html
<!DOCTYPE html>
<html>

<body>

    <h1 id="title">Hello</h1>

    <script src="script.js"></script>

</body>

</html>
```

**script.js**

```javascript
const heading = document.getElementById("title");

heading.textContent = "Welcome";
```

Output

```
Hello

↓

Welcome
```

```

↓

↓

Creates DOM

↓

↓

↓

```

# Common Mistakes

## 1. Forgetting `<script>`

Wrong

```html
console.log("Hello");
```

Correct

```html
<script>

console.log("Hello");

</script>
```

---

## 2. Wrong File Path

```html
<script src="app.js"></script>
```

If `app.js` doesn't exist,

```
404 Error
```

JavaScript won't load.

---

## 3. Accessing HTML Before It Exists

Wrong

```html
<head>

<script>

document.getElementById("title");

</script>

</head>
```

The element hasn't been created yet.

Correct

```html
<body>

...

<script>

document.getElementById("title");

</script>

</body>
```

or

```html
<script src="script.js" defer></script>
```

---

## 4. Putting Code Inside `<script src="">`

Wrong

```html
<script src="script.js">

console.log("Hello");

</script>
```

Only the external file is executed.

# Interview Questions

## Q1. What are the three ways to add JavaScript to HTML?

- Inline JavaScript
- Internal JavaScript
- External JavaScript

---

## Q2. Which method is recommended?

**External JavaScript**, because it improves readability, maintainability, reuse, and browser caching.

---

## Q3. Where should you place the `<script>` tag?

Either:

- At the end of the `<body>`, or
- In the `<head>` with the `defer` attribute.

---

## Q4. Difference between `defer` and `async`?

- `defer` waits until the HTML is fully parsed and preserves script order.
- `async` executes as soon as the download finishes and does not preserve order.

---

# Summary

| Method | Description |
|---------|-------------|
| Inline | JavaScript inside HTML attributes |
| Internal | JavaScript inside `<script>` tags |
| External | JavaScript in a separate `.js` file |
| `defer` | Executes after HTML parsing |
| `async` | Executes immediately after download |

---
