# Functions in JavaScript

> **A function is a reusable block of code designed to perform a specific task.**

Think of a function as a **machine**.

```
Input
   │
   ▼
Function
   │
   ▼
Output
```

Example

```javascript
function add(a, b) {
    return a + b;
}

console.log(add(10, 20));
```

Output

```
30
```

---

# Why Do We Need Functions?

Without functions

```javascript
let a = 10;
let b = 20;
console.log(a + b);

a = 30;
b = 40;
console.log(a + b);

a = 50;
b = 60;
console.log(a + b);
```

Repeated code.

---

Using functions

```javascript
function add(a, b) {
    return a + b;
}

console.log(add(10,20));
console.log(add(30,40));
console.log(add(50,60));
```

Advantages

- Code Reusability
- Less Duplication
- Easy Maintenance
- Better Readability

# What Does return Do?

`return` performs **two actions**.

## 1. Returns a value

```javascript
return 10;
```

---

## 2. Stops function execution

```javascript
function test(){

    console.log("A");

    return;

    console.log("B");

}

test();
```

Output

```
A
```

`"B"` never executes.

---

let value = test();

If you don't return anything,

# Arrow Function

Introduced in ES6.

```javascript
const greet = () => {

    console.log("Hello");

};

greet();
```

Output

```
Hello
```

---

Arrow Function with Return

```javascript
const add = (a,b) => {

    return a+b;

};

console.log(add(10,20));
```

Output

```
30
```

---

Short Arrow Function

```javascript

```

Equivalent

```javascript

    return a+b;

}
```

# Function Hoisting

Function Declaration

```javascript
greet();

function greet(){

    console.log("Hello");

}
```

Output

```
Hello
```

Reason

Entire function is hoisted.

---

Function Expression

```javascript
greet();

var greet=function(){

};
```

Output

```
TypeError
```

Reason

Only the variable is hoisted.

---

Arrow Function

```javascript

const greet=()=>{

};
```

Output

```
ReferenceError
```

Reason

`const` is in the TDZ.

# First-Class Functions

JavaScript treats functions like values.

Functions can

- Be stored in variables
- Be passed as arguments
- Be returned from another function

Example

```javascript
const greet=function(){

};

function execute(fn){

    fn();

}

execute(greet);
```

# Summary

| Concept | Description |
|----------|-------------|
| Function | Reusable block of code |
| Parameters | Variables in function definition |
| Arguments | Values passed during function call |
| Return | Sends value back and stops execution |
| Function Declaration | Fully hoisted |
| Function Expression | Stored in variable |
| Arrow Function | Short syntax introduced in ES6 |
| Callback | Function passed to another function |
| Higher-Order Function | Takes/returns functions |
| Recursive Function | Calls itself |
| IIFE | Executes immediately |
| Pure Function | Same input → Same output |
| First-Class Function | Functions behave like values |

---

# Golden Rules

- A function **executes only when it is called**.
- **Parameters** receive values; **arguments** provide values.
- `return` both **returns a value** and **ends function execution**.
- Function declarations are fully hoisted; function expressions and arrow functions are not.
- Functions are **first-class citizens** in JavaScript, meaning they can be stored, passed, and returned like any other value.
