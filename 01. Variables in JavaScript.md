# Variables in JavaScript

> "A variable is not a box that stores data. It is a **named reference** to a value in memory."

---

# What is a Variable?

A variable is an identifier (name) given to a memory location so that we can store, access, and manipulate data.

Instead of remembering memory addresses, we use meaningful names.

```javascript
let age = 20;
```

Here,

- `age` â†’ Variable name (identifier)
- `20` â†’ Value stored
- `=` â†’ Assignment operator

---

# Why Do We Need Variables?

Without variables:

```javascript
console.log(10 + 20);
console.log(10 + 20);
console.log(10 + 20);
```

If the value changes:

```javascript
10 â†’ 15
```

You must modify every occurrence.

Using variables:

```javascript
let price = 10;

console.log(price + 20);
console.log(price + 20);
console.log(price + 20);
```

Only one change is required.

```javascript
price = 15;
```

# Naming Rules

Valid

```javascript
let age;
let firstName;
let _name;
let $price;
let student1;
```

Invalid

```javascript
let 1name;
let first-name;
let let;
```

Reason

- Cannot start with a number.
- Cannot contain spaces.
- Cannot use keywords.
- Hyphen is treated as the subtraction operator.

# Types of Variable Declarations

JavaScript provides three ways to declare variables.

```javascript
var
let
const
```

---

# var

```javascript
var age = 20;
```

Characteristics

- Function scoped
- Can be redeclared
- Can be reassigned
- Hoisted
- Initialized with `undefined`

Example

```javascript
var x = 10;
var x = 20;

console.log(x);
```

Output

```
20
```

---

# let

```javascript
let age = 20;
```

Characteristics

- Block scoped
- Cannot be redeclared
- Can be reassigned
- Hoisted
- Exists in the Temporal Dead Zone until initialized

Example

```javascript
let age = 20;

age = 30;
```

Output

```
30
```

---

# const

```javascript
const PI = 3.14;
```

Characteristics

- Block scoped
- Cannot be redeclared
- Cannot be reassigned
- Must be initialized during declaration
- Exists in the Temporal Dead Zone until initialized

Example

```javascript
const PI = 3.14;
```

Many think



Output


# Interview Questions

### Q1. Difference between declaration and initialization?

**Declaration**

Creating the variable.

```javascript
let age;
```

**Initialization**

Giving it its first value.

```javascript
age = 20;
```

---

### Q2. Why does `var` print `undefined`?

Because `var` is hoisted and initialized with `undefined` when its scope is entered.

---

### Q3. Why do `let` and `const` throw `ReferenceError` before declaration?

Because they are in the **Temporal Dead Zone (TDZ)** until they are initialized.

---

### Q4. Which should you use in modern JavaScript?

- `const` â†’ Default choice.
- `let` â†’ When reassignment is needed.
- `var` â†’ Only for maintaining legacy code.

# Golden Rules

- A variable is a **named reference** to a value.
- `var` is hoisted and initialized with `undefined`.
- `let` and `const` are hoisted but remain uninitialized until execution reaches their declaration.
- The period before initialization for `let` and `const` is called the **Temporal Dead Zone (TDZ)**.
- Use **`const` by default**, **`let` when reassignment is required**, and **avoid `var`** in modern JavaScript.
