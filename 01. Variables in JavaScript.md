# Variables in JavaScript

> **A variable is a named reference to a value. It gives a readable name to a memory location so the program can access and manipulate data.**

---

# Why Variables Exist

Without variables, every value must be written repeatedly.

```javascript
console.log(10 + 20);
console.log(10 + 20);
console.log(10 + 20);
```

Changing `10` to `15` requires editing every occurrence.

Variables eliminate repetition.

```javascript
let price = 10;

console.log(price + 20);
console.log(price + 20);
console.log(price + 20);

price = 15;
```

Only one value changes.

---

# Variable Anatomy

```javascript
let age = 20;
```

```
let   age   =   20
│      │        │
│      │        └── Value
│      └────────── Identifier (Variable Name)
└───────────────── Declaration Keyword
```

---

# Variable Life Cycle

Every variable goes through three stages.

## 1. Declaration

The variable is created.

```javascript
let age;
```

---

## 2. Initialization

The variable receives its first value.

```javascript
let age = 20;
```

---

## 3. Reassignment

The value changes.

```javascript
age = 30;
```

---

# Declaration vs Initialization

```javascript
let age;      // Declaration

age = 20;     // Initialization

age = 30;     // Reassignment
```

---

# Variable Naming Rules

Valid

```javascript
let age;
let firstName;
let _count;
let $price;
let student1;
```

Invalid

```javascript
let 1age;
let first-name;
let let;
```

Rules

- Cannot start with a number.
- Cannot contain spaces.
- Cannot use reserved keywords.
- `-` is subtraction, not part of a variable name.

---

# Naming Convention

Prefer meaningful names.

Good

```javascript
let totalPrice;
let firstName;
let employeeSalary;
```

Poor

```javascript
let a;
let x;
let abc;
```

Code is read far more often than it is written.

---

# Ways to Declare Variables

JavaScript provides three keywords.

```javascript
var
let
const
```

Each differs in **scope**, **hoisting**, **reassignment**, and **redeclaration**.

---

# var

```javascript
var age = 20;
```

Properties

- Function scoped
- Can be redeclared
- Can be reassigned
- Hoisted
- Automatically initialized with `undefined`

```javascript
var x = 10;
var x = 20;

console.log(x);
```

Output

```
20
```

---

# let

```javascript
let age = 20;
```

Properties

- Block scoped
- Cannot be redeclared
- Can be reassigned
- Hoisted
- Exists in the Temporal Dead Zone until initialized

```javascript
let age = 20;

age = 30;
```

---

# const

```javascript
const PI = 3.14;
```

Properties

- Block scoped
- Cannot be redeclared
- Cannot be reassigned
- Must be initialized immediately
- Exists in the Temporal Dead Zone until initialized

---

# Memory Creation Phase

Before executing any code, JavaScript scans the scope and allocates memory.

```javascript
var a = 10;
let b = 20;
const c = 30;
```

Memory after entering the scope

```
a → undefined

b → <uninitialized>

c → <uninitialized>
```

Key idea

- `var` receives `undefined`.
- `let` and `const` exist but cannot be accessed.

---

# Execution Phase

JavaScript executes code from top to bottom.

```javascript
var a = 10;
```

```
a → 10
```

```javascript
let b = 20;
```

```
b → 20
```

```javascript
const c = 30;
```

```
c → 30
```

---

# Hoisting

**Hoisting is JavaScript's behavior of creating variables before execution begins.**

The difference is **initialization**, not creation.

## var

```javascript
console.log(a);

var a = 10;
```

Output

```
undefined
```

Reason

```
Created ✔
Initialized ✔ (undefined)
Accessible ✔
```

---

## let

```javascript
console.log(a);

let a = 10;
```

Output

```
ReferenceError
```

Reason

```
Created ✔
Initialized ✖
Accessible ✖
```

---

## const

```javascript
console.log(a);

const a = 10;
```

Output

```
ReferenceError
```

Same reason as `let`.

---

# Temporal Dead Zone (TDZ)

> **The Temporal Dead Zone is the time between entering a scope and initializing a `let` or `const` variable.**

Timeline

```
Enter Scope
      │
      ▼
Variable Created
      │
      ▼
TDZ
      │
      ▼
Initialization
      │
      ▼
Variable Accessible
```

Example

```javascript
{
    console.log(age);

    let age = 20;
}
```

Output

```
ReferenceError
```

# const and Objects

`const` protects the **reference**, not the object.

```javascript
const user = {
    name: "Madan"
};

user.name = "John";
```

✔ Allowed

The object changed.

The reference did not.

---

```javascript
const user = {
    name: "Madan"
};

user = {};
```

✖ TypeError

The reference changed.

---

# Best Practices

Use `const` unless the value must change.

```javascript
const PI = 3.14;
```

Use `let` when reassignment is required.

```javascript
let count = 0;

count++;
```

Avoid `var` in modern JavaScript.

---

# Common Mistakes

### Accessing `let` before initialization

```javascript
console.log(age);

let age = 20;
```

Result

```
ReferenceError
```

---

### Declaring `const` without a value

```javascript
const age;
```

Result

```
SyntaxError
```

Correct

```javascript
const age = 20;
```

---

### Reassigning a `const`

```javascript
const user = {};

user = {};
```

Result

```
TypeError
```

# Interview Questions

### What is the difference between declaration and initialization?

Declaration creates the variable.

```javascript
let age;
```

Initialization gives it its first value.

```javascript
age = 20;
```

---

### Why does `var` print `undefined`?

Because it is initialized with `undefined` during the memory creation phase.

---

### Why do `let` and `const` throw `ReferenceError`?

Because they are inside the **Temporal Dead Zone** until initialization.

---

### Which keyword should be used today?

- `const` → Default choice
- `let` → When reassignment is needed
- `var` → Legacy code only

# Comparison

| Feature | var | let | const |
|---------|-----|-----|--------|
| Scope | Function | Block | Block |
| Hoisted | ✅ | ✅ | ✅ |
| Initialized During Memory Creation | ✅ (`undefined`) | ❌ | ❌ |
| Temporal Dead Zone | ❌ | ✅ | ✅ |
| Redeclaration | ✅ | ❌ | ❌ |
| Reassignment | ✅ | ✅ | ❌ |
| Initialization Required | ❌ | ❌ | ✅ |
| Modern Usage | Avoid | Yes | Preferred |

# Key Takeaways

- A variable is a **named reference**, not a storage box.
- Every variable is **created before execution**.
- `var` is immediately initialized with `undefined`.
- `let` and `const` stay in the **Temporal Dead Zone** until initialized.
- `const` prevents changing the reference, **not** mutating the object it references.
- Prefer `const`, use `let` only when reassignment is necessary, and avoid `var` in modern JavaScript.
