# Arrays in JavaScript

> **An array is an ordered collection of values stored under a single variable, where each value is accessed using its index.**

Think of an array as a **numbered shelf**.

```
Index

0    1    2    3
│    │    │    │
▼    ▼    ▼    ▼

10   20   30   40
```

Instead of creating many variables,

```javascript
let mark1 = 90;
let mark2 = 85;
let mark3 = 95;
let mark4 = 80;
```

we write

```javascript
const marks = [90, 85, 95, 80];
```

---

# Why Do We Need Arrays?

Suppose you have marks of 100 students.

Without arrays

```javascript
let mark1 = 90;
let mark2 = 85;
let mark3 = 95;
...
let mark100 = 78;
```

Impossible to manage.

With arrays

```javascript
const marks = [
    90,
    85,
    95,
    80
];
```

One variable stores multiple values.

---

# What is an Array Internally?

An array is a **special type of object**.

```javascript
const numbers = [10,20,30];
```

Internally JavaScript stores something like

```
numbers

↓

{

0 : 10,

1 : 20,

2 : 30,

length : 3

}
```

That's why

```javascript
typeof numbers
```

returns

```
object
```

# Array Reference

Arrays are reference types.

```javascript
const a = [
    1,
    2
];

const b = a;
```

Memory

```
a

↓

Array

↑

b
```

Changing

```javascript
b[0] = 100;
```

Now

```javascript
console.log(a);
```

Output

```
[
100,
2
]
```

Both variables refer to the same array.

---

# Copying Arrays

Wrong

```javascript
const copy = original;
```

Both point to the same array.

Correct

```javascript
const copy = [
    ...original
];
```

or

```javascript
const copy = original.slice();
```

# Common Mistakes

## Mistake 1

```javascript
arr[100]
```

Returns

```
undefined
```

Not an error.

---

## Mistake 2

```javascript
typeof []
```

Output

```
object
```

Arrays are objects internally.

---

## Mistake 3

```javascript
const arr = [1,2];

arr = [3,4];
```

Output

```
TypeError
```

Reason

`const` prevents changing the reference.

But

```javascript
arr.push(3);
```

is allowed.

---

# Golden Rules

- Arrays store **multiple values** in a single variable.
- Array indexing **always starts at `0`**.
- Arrays are **special objects** in JavaScript.
- Arrays are **reference types**, so assigning one array to another copies the reference, not the contents.
- Use `Array.isArray()` to check if a value is an array.
- Use `slice()` when you need a copy without modifying the original array, and `splice()` when you intentionally want to modify it.
- `const` arrays **can be modified**, but the array variable **cannot be reassigned** to a different array.
