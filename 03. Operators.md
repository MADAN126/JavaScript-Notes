# JavaScript Operators

> **Operators are special symbols that perform operations on one or more operands (values or variables).**

Example

```javascript
let sum = 10 + 20;
```

Here,

- `10` → Operand
- `20` → Operand
- `+` → Operator

```
10 + 20
│    │
│    └── Operand
│
└────── Operand

+

↓

Operator
```

---

# Why Do We Need Operators?

Imagine JavaScript without operators.

Instead of

```javascript
10 + 20
```

you would need a function for everything.

Operators make code

- Short
- Readable
- Easy to understand

---

# 1. Arithmetic Operators

Used for mathematical calculations.

| Operator | Meaning |
|----------|---------|
| + | Addition |
| - | Subtraction |
| * | Multiplication |
| / | Division |
| % | Modulus |
| ** | Exponentiation |

---

## Addition (+)

```javascript
let a = 10;
let b = 20;

console.log(a + b);
```

Output

```
30
```

---

## Subtraction (-)

```javascript
console.log(20 - 5);
```

Output

```
15
```

---

## Multiplication (*)

```javascript
console.log(5 * 4);
```

Output

```
20
```

---

## Division (/)

```javascript
console.log(20 / 5);
```

Output

```
4
```

---

## Modulus (%)

Returns the **remainder**, not the quotient.

```javascript
console.log(10 % 3);
```

Output

```
1
```

Example

```
10 ÷ 3

Quotient = 3

Remainder = 1
```

Used for

- Even/Odd checking
- Cyclic logic
- Pagination

Example

```javascript
console.log(8 % 2);
```

Output

```
0
```

Even number.

---

## Exponentiation (**)

```javascript
console.log(2 ** 3);
```

Output

```
8
```

Means

```
2 × 2 × 2
```

# 3. Comparison (Relational) Operators

Used to compare values.

Always returns

```
true

or

false
```

---

## >

```javascript
10 > 5
```

Output

```
true
```

---

## <

```javascript
3 < 2
```

Output

```
false
```

---

## >=

```javascript
10 >= 10
```

Output

```
true
```

---

## <=

```javascript
5 <= 3
```

Output

```
false
```

---

## == (Loose Equality)

Compares only values.

Performs type coercion.

```javascript
10 == "10"
```

Output

```
true
```

Because

```
"10"

↓

10
```

---

## === (Strict Equality)

Compares

- Value
- Type

```javascript
10 === "10"
```

Output

```
false
```

Types are different.

Always prefer

```
===
```

---

## !=

```javascript
10 != "10"
```

Output

```
false
```

Because loose comparison converts types.

---

## !==

```javascript
10 !== "10"
```

Output

```
true
```

Different types.

---

# 4. Logical Operators

Used to combine conditions.

---

## AND (&&)

Returns true only if **all conditions** are true.

```javascript
true && true
```

Output

```
true
```

---

```javascript
true && false
```

Output

```
false
```

Example

```javascript
let age = 22;
let citizen = true;

console.log(age >= 18 && citizen);
```

Output

```
true
```

---

## OR (||)

Returns true if **at least one condition** is true.

```javascript
false || true
```

Output

```
true
```

---

## NOT (!)

Reverses the boolean value.

```javascript
console.log(!true);
```

Output

```
false
```

---

# Short-Circuit Evaluation

## AND

```javascript
false && console.log("Hello");
```

Output

```
false
```

`console.log()` never runs.

Reason

```
false && anything

↓

Always false
```

---

## OR

```javascript
true || console.log("Hello");
```

Output

```
true
```

Second operand is skipped.

# 6. Increment & Decrement

---

## ++

```javascript
let x = 5;

x++;

console.log(x);
```

Output

```
6
```

---

## --

```javascript
let x = 5;

x--;

console.log(x);
```

Output

```
4
```

---

## Prefix

```javascript
let x = 5;

console.log(++x);
```

Output

```
6
```

Increment first.

Return later.

---

## Postfix

```javascript
let x = 5;

console.log(x++);
```

Output

```
5
```

Increment happens after returning.

Final value

```
6
```

---

# 7. Conditional (Ternary) Operator

Syntax

```javascript
condition ? value1 : value2
```

Example

```javascript
let age = 20;

let result = age >= 18 ? "Adult" : "Minor";
```

Output

```
Adult
```

Equivalent

```javascript
if(age >= 18){

    result = "Adult";

}else{

    result = "Minor";

}
```

---

# 8. Nullish Coalescing (??)

Returns the right value only when the left value is

- null
- undefined

Example

```javascript
let name = null;

console.log(name ?? "Guest");
```

Output

```
Guest
```

---

Difference from ||

```javascript
console.log(0 || 100);
```

Output

```
100
```

Because 0 is falsy.

---

```javascript
console.log(0 ?? 100);
```

Output

```
0
```

Because 0 is not null or undefined.

---

# 9. Optional Chaining (?.)

Safely accesses nested properties.

Without

```javascript
user.address.city
```

If address doesn't exist

```
TypeError
```

With

```javascript
user?.address?.city
```

Output

```
undefined
```

No error.

# Operator Precedence

Example

```javascript
10 + 2 * 3
```

Output

```
16
```

Not

```
36
```

Reason

```
*

↓

+

```

Multiplication has higher precedence.

Use parentheses for clarity.

```javascript
(10 + 2) * 3
```

Output

```
36
```

# Common Mistakes

## Mistake 1

```javascript
if(a = 10)
```

Wrong.

Assignment.

Correct

```javascript
if(a == 10)
```

or better

```javascript
if(a === 10)
```

---

## Mistake 2

```javascript
10 == "10"
```

Returns

```
true
```

Prefer

```javascript
10 === "10"
```

Returns

```
false
```

---

## Mistake 3

```javascript
console.log(0 || 100);
```

Returns

```
100
```

Use

```javascript
0 ?? 100
```

when you only want to handle `null` or `undefined`.

# Interview Questions

### Q1

Difference between `==` and `===`?

- `==` compares values after type coercion.
- `===` compares both value and type.

---

### Q2

Difference between `||` and `??`?

- `||` treats all falsy values as missing.
- `??` treats only `null` and `undefined` as missing.

---

### Q3

Difference between prefix and postfix increment?

```javascript
++x
```

Increment first.

Return later.

```javascript
x++
```

Return first.

Increment later.

# Golden Rules

- Operators work on **operands** (values or variables).
- Prefer `===` over `==`.
- Use `??` instead of `||` when `0`, `false`, or `""` are valid values.
- Understand the difference between **prefix** and **postfix** increment.
- Use parentheses `()` when operator precedence might make expressions unclear.
