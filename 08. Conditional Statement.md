# Conditional Statements in JavaScript

> **Conditional statements allow JavaScript to make decisions by executing different blocks of code based on whether a condition is `true` or `false`.**

Think of them like traffic signals.

```
Condition

      │
      ▼

 Is it true?
   /      \
 Yes      No
 │         │
 ▼         ▼

Execute   Execute
Block A   Block B
```

Without conditional statements, a program would execute every line of code from top to bottom without making any decisions.

---

# Why Do We Need Conditional Statements?

Imagine a login system.

Without conditions

```javascript
console.log("Welcome");
```

Every user would be welcomed, even if the password is wrong.

Using conditions

```javascript
if(password === "1234"){

    console.log("Welcome");

}
```

Now JavaScript makes a decision.

# Types of Conditional Statements

JavaScript provides four conditional statements.

```
Conditional Statements

│

├── if

├── if...else

├── if...else if...else

└── switch
```

# Boolean Expressions

Every condition must evaluate to

```
true

or

false
```

Example

```javascript
10 > 5
```

Result

```
true
```

Example

```javascript
20 < 10
```

Result

```
false
```

# if Statement

Executes a block only if the condition is true.

Syntax

```javascript
if(condition){

    // code

}
```

Example

```javascript
let age = 20;

if(age >= 18){

    console.log("Eligible to vote");

}
```

Output

```
Eligible to vote
```

---

If condition is false

```javascript
let age = 15;

if(age >= 18){

    console.log("Eligible");

}
```

Output

```
(No Output)
```

The block is skipped.

---

```
Condition

↓

true ?

↓

Yes

↓

Execute Block

↓

Continue Program

-------------------------

Condition

↓

false ?

↓

Skip Block

↓

```

# if...else Statement

Executes one block if true and another block if false.

Syntax

```javascript
if(condition){

    // true block

}else{

    // false block

}
```

Example

```javascript
let age = 15;

if(age >= 18){

    console.log("Adult");

}else{

    console.log("Minor");

}
```

Output

```
Minor
```

---

Flow

```
Condition

↓

true ?

↓

Yes

↓

if Block

----------------

No

↓

else Block
```

---

# if...else if...else

Used when there are multiple conditions.

Syntax

```javascript
if(condition1){

}

else if(condition2){

}

else if(condition3){

}

else{

}
```

Example

```javascript
let marks = 82;

if(marks >= 90){

    console.log("Grade A");

}

else if(marks >= 75){

    console.log("Grade B");

}

else if(marks >= 50){

    console.log("Grade C");

}

else{

    console.log("Fail");

}
```

Output

```
Grade B
```

---

# How JavaScript Executes

JavaScript checks conditions **from top to bottom**.

As soon as one condition becomes true,

it executes that block

and **skips all remaining conditions**.

Example

```javascript
let number = 100;

if(number > 0){

    console.log("Positive");

}

else if(number > 50){

    console.log("Greater than 50");

}
```

Output

```
Positive
```

The second condition is never checked.

---

# Nested if

An if statement inside another if statement.

```javascript
let age = 20;

let citizen = true;

if(age >= 18){

    if(citizen){

        console.log("Can Vote");

    }

}
```

Output

```
Can Vote
```

# Logical Operators with Conditions

## AND (&&)

Both conditions must be true.

```javascript
let age = 20;

let citizen = true;

if(age >= 18 && citizen){

    console.log("Eligible");

}
```

Output

```
Eligible
```

---

## OR (||)

At least one condition must be true.

```javascript
let isAdmin = false;

let isManager = true;

if(isAdmin || isManager){

    console.log("Access Granted");

}
```

Output

```
Access Granted
```

---

## NOT (!)

Reverses the result.

```javascript
let loggedIn = false;

if(!loggedIn){

    console.log("Please Login");

}
```

Output

```
Please Login"
```

# Truthy and Falsy Values

JavaScript automatically converts values to `true` or `false` in conditions.

Falsy values

```javascript
false
0
-0
0n
""
null
undefined
NaN
```

Everything else is **truthy**.

Example

```javascript
if("Hello"){

    console.log("Runs");

}
```

Output

```
Runs
```

---

Example

```javascript
if(0){

    console.log("Runs");

}
```

Output

```
(No Output)
```

---

# Ternary Operator

A shorter form of `if...else`.

Syntax

```javascript
condition ? value1 : value2
```

Example

```javascript
let age = 20;

let result = age >= 18 ? "Adult" : "Minor";

console.log(result);
```

Output

```
Adult
```

Equivalent

```javascript
if(age >= 18){

    result = "Adult";

}else{

    result = "Minor";

}
```

---

# switch Statement

Used when comparing one value against multiple cases.

Syntax

```javascript
switch(value){

    case value1:

        break;

    case value2:

        break;

    default:

}
```

Example

```javascript
let day = 2;

switch(day){

    case 1:

        console.log("Monday");

        break;

    case 2:

        console.log("Tuesday");

        break;

    default:

        console.log("Invalid");

}
```

Output

```
Tuesday
```

---

# Why break is Important

Without break

```javascript
let day = 1;

switch(day){

    case 1:

        console.log("Monday");

    case 2:

        console.log("Tuesday");

}
```

Output

```
Monday

Tuesday
```

Execution falls through.

Correct

```javascript
case 1:

    console.log("Monday");

    break;
```

---

# if vs switch

Use **if**

- Ranges
- Multiple conditions
- Logical operators

Example

```javascript
if(age >= 18)
```

---

Use **switch**

- Exact value matching

Example

```javascript
switch(color)
```

# Common Mistakes

## Using = instead of == or ===

Wrong

```javascript
if(age = 18)
```

Assignment happens.

Correct

```javascript
if(age === 18)
```

---

## Forgetting break

Wrong

```javascript
case 1:

console.log("One");
```

May execute the next case.

---

## Using ==

Avoid

```javascript
if(age == "18")
```

Prefer

```javascript
if(age === 18)
```

---

## Incorrect Order

Wrong

```javascript
let marks = 95;

if(marks >= 50){

    console.log("Pass");

}

else if(marks >= 90){

    console.log("A");
}
```

Output

```
Pass
```

Correct

```javascript
if(marks >= 90){

    console.log("A");

}

else if(marks >= 50){

    console.log("Pass");

}
```

Always place **more specific conditions before broader ones**.

# Interview Questions

## Q1

Difference between `if` and `switch`?

| if | switch |
|----|---------|
| Supports ranges | Exact value matching |
| Can use logical operators | Cannot use ranges directly |
| More flexible | Cleaner for many fixed values |

---

## Q2

Difference between `if` and ternary?

`if`

```javascript
if(age >= 18){

}
```

For complex logic.

---

Ternary

```javascript
age >= 18 ? "Adult" : "Minor"
```

For simple value selection.

---

## Q3

Why is `break` needed in switch?

To stop execution after a matching case.

Without it, JavaScript continues into the next cases (**fall-through**).

---

## Q4

What are truthy and falsy values?

Falsy values

```
false
0
-0
0n
""
null
undefined
NaN
```

Everything else is truthy.

# Summary

| Statement | Purpose |
|-----------|----------|
| `if` | Execute code if condition is true |
| `if...else` | Choose between two blocks |
| `if...else if...else` | Multiple conditions |
| `switch` | Compare one value with many cases |
| Ternary (`?:`) | Short form of `if...else` |

---

# Golden Rules

- Every conditional statement evaluates a **Boolean expression** (`true` or `false`).
- `if` is best for **ranges and complex conditions**.
- `switch` is best for **exact value comparisons**.
- Always use `===` instead of `==` unless type coercion is intentional.
- In an `if...else if...else` chain, JavaScript stops checking as soon as it finds the **first true condition**.
- Always use `break` in `switch` unless you intentionally want **fall-through**.
- Use the **ternary operator** only for simple two-way decisions, not complex logic.
- Remember the falsy values: `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, and `NaN`.
