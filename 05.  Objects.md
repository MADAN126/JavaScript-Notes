# Objects in JavaScript

> **An object is a collection of related data and behavior stored as `key : value` pairs.**

Think of an object as a **real-world entity**.

A student has:

- Name
- Age
- Department
- Marks

Instead of creating separate variables,

```javascript
let name = "Madan";
let age = 22;
let dept = "CSE";
```

we group everything together.

```javascript
const student = {
    name: "Madan",
    age: 22,
    dept: "CSE"
};
```

Now all related information belongs to one object.

---

# Why Do We Need Objects?

Without objects

```javascript
let name = "Madan";
let age = 22;
let city = "Chennai";
let phone = "9876543210";
```

Imagine storing 100 students.

You'll need

```
name1
age1
city1

name2
age2
city2

...

name100
age100
city100
```

Very difficult.

Using objects

```javascript
const student = {
    name: "Madan",
    age: 22,
    city: "Chennai"
};
```

Everything is organized.

There are two ways.


## Bracket Notation


# Difference

Dot notation

```javascript
student.name
```

Bracket notation

```javascript
student["name"]
```

Bracket notation is useful when

- Property name is stored in a variable
- Property contains spaces
- Property starts with numbers

Example

```javascript
const key = "age";

console.log(student[key]);
```

Output

```
22
```

# Methods

Functions inside objects are called methods.

```javascript
const student = {

    name:"Madan",

    greet:function(){

        console.log("Hello");

    }

};

student.greet();
```

Output

```
Hello
```

---

# Shorthand Method Syntax

Instead of

```javascript
const student = {

    greet:function(){

        console.log("Hello");

    }

};
```

Write

```javascript
const student = {

    greet(){

        console.log("Hello");

    }

};
```

Both are the same.

---

# this Keyword

Inside an object,

`this` refers to the current object.

```javascript
const student = {

    name:"Madan",

    greet(){

        console.log(this.name);

    }

};

student.greet();
```

Output

```
Madan
```

# for...in Loop

Used to iterate over object properties.

```javascript
const student = {

    name:"Madan",

    age:22

};

for(let key in student){

    console.log(key);

}
```

Output

```
name

age
```

Getting values

```javascript
for(let key in student){

    console.log(student[key]);

}
```

Output

```
Madan

22
```

# Object Reference

Objects are stored by reference.

```javascript
const user1 = {

    name:"Madan"

};

const user2 = user1;
```

Memory

```
user1
   │
   ▼
Object

▲
│

user2
```

Changing

```javascript
user2.name = "John";
```

Now

```javascript
console.log(user1.name);
```

Output

```
John
```

Both variables point to the same object.

# Object Comparison

```javascript
const a = {

    name:"Madan"

};

const b = {

    name:"Madan"

};

console.log(a == b);
```

Output

```
false
```

Reason

Different memory addresses.

---

```javascript
const a = {

    name:"Madan"

};

const b = a;

console.log(a == b);
```

Output

```
true
```

Both point to the same object.

# Object.freeze()

Makes an object immutable.

```javascript
const student = {

    name:"Madan"

};

Object.freeze(student);

```

Output

```

```

Change ignored.

# Object.seal()

Allows updating existing properties.

Prevents

- Adding new properties
- Deleting properties

Example

```javascript

```

# Common Mistakes

## Using Dot Notation with Variable

Wrong

```javascript
const key = "age";

student.key
```

Output

```
undefined
```

Correct

```javascript
student[key]
```

---

## Comparing Objects

```javascript
{} == {}
```

Output

```
false
```

Objects compare references, not contents.

---

## Thinking const Makes Objects Immutable

Wrong

```javascript
const student = {

    age:22

};

student.age = 30;
```

Allowed.

Only the reference is constant.

---

# Golden Rules

- An object is a **collection of key-value pairs**.
- Objects are **reference types**, not primitive values.
- Use **dot notation** for fixed property names and **bracket notation** for dynamic ones.
- Objects are **mutable**, even when declared with `const`; `const` prevents changing the reference, not the object's contents.
- Functions stored in objects are called **methods**.
- Two objects with identical contents are **not equal** unless they reference the same object in memory.
- Use `Object.keys()`, `Object.values()`, and `Object.entries()` to inspect object properties.
